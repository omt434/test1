"Write Once, Run Anywhere" (WORA)

This is the main golden design rule of Java! Program compiles into intermediate code (bytecode) which can execute on ANY OS with a JVM.

📌 1. What is Java? (Exam Definition)

Java is a high-level, object-oriented, class-based, secure, robust, and platform-independent programming language.

Originally designed for smart electronic devices (like interactive TVs), but it took off to become the ultimate choice for internet-ready applications, Android apps, and massive enterprise architectures.

VIVA CRITICAL!

📒 Core Characteristics:

  • High-Level (No direct hardware assembly needed)
  • Object-Oriented (Encapsulation, Class, Poly, inheritance)
  • Platform Independent
  • Distributed & Multi-threaded
  • Highly Secure & Extremely Robust

📜 Short History:

💻 Developed in 1991 by the famous James Gosling & his core team at Sun Microsystems.

💡 Originally named OAK (after an oak tree outside Gosling's cabin window) & subsequently renamed to JAVA in 1995 (from Java Espresso!).

🕒 Historical Milestone Ledger:

❓ Why was Java needed?

Before Java: Program compiled straight into CPU-specific machine instructions. A program written on Windows could NEVER run on Unix without rewriting or re-compiling source code!

Java Solution: Introduced the virtual processor layer (JVM). This decoupled program execution from the physical hardware host machine entirely!

Features of Java

"The Buzzwords of Java" - High scoring in 10-Mark Exams!

🎨 Mind Map: 9 Essential Java Buzzwords

1. Simple 2. Object-Oriented 3. Platform Independent 4. Secure 5. Robust 6. Portable 7. Multithreaded 8. Distributed 9. High Performance

1. Simple 📝

Syntax is directly clean and based on simplified C++ without complex, confusing features like pointers, operator overloading, multiple inheritance, or manual memory freeing!

2. Object-Oriented (OOP) 📦

Everything in Java is written inside a class as objects! Models real-world entities through class hierarchies, encapsulation, data hiding, and polymorphic interfaces.

3. Platform Independent 💻

Unlike C/C++ which compile to native OS binaries, Java compiles into bytecode (.class file) which can run anywhere with a JVM installed.

4. Highly Secure 🛡️

No explicit memory pointers, JVM verifies incoming bytecodes, and security managers run code inside sandbox wrappers to block virus files or disk intrusion!

5. Robust & Tough 💪

"Robust" translates to exceptionally reliable. It achieves extreme stability via strong garbage collection, fully runtime-enforced strict typechecking, and elegant exception handling routines!

6. Portable 🎒

No hardware-specific implementation properties (e.g. integer type is guaranteed 4 bytes size on Windows, UNIX, and MacOS alike!). Take bytecode anywhere effortlessly.

7. Multithreaded 🧵

Enables programs to carry out several paths of execution concurrently. Extremely useful for processing web requests or running games!

8. High Performance ⚡

By using JIT (Just-In-Time) compilation alongside native code generation, execution speed has reached nearly native speeds today!

JDK vs. JRE vs. JVM

This is the single most common university question! Study closely.

🔍 Concentric Architectural Stack: (Hover over blocks)

Outer Box
☕ JDK (Java Development Kit) = JRE + Tools

Used to WRITE & compile Java programs (includes javac, jdb)

Middle Box
📦 JRE (Java Runtime Environment) = JVM + Core Libs

Environment required to RUN compiled Java programs (no developer tools)

Core Engine
⚙️ JVM (Java Virtual Machine)

Pure abstract engine. Loads, verifies, interprets, & runs bytecode.

📐 Dynamic Equation 1:

JRE = JVM + Core Class Libraries

📐 Dynamic Equation 2:

JDK = JRE + Development Kit Tools

Java Program Architecture

How exactly does a Java program compile and run? Step-by-step breakdown.

🚶‍♂️ Detailed Compile & Execution Steps

📁 Step 1: Write Source Code

We write instructions in simple plain text and save with a .java suffix (e.g. Demo.java).

⚙️ Step 2: Compile (The javac machine)

The developer compiles the program using terminal command: javac Demo.java. The compiler outputs a platform-neutral intermediate code file.

💿 Step 3: Bytecode Generation

A new machine-neutral binary is auto-created with a .class suffix (e.g., Demo.class).

⚡ Step 4: Run via JVM Interpreter

The class file is loaded into memory, verified, and run using command: java Demo. JVM converts the binary blocks into OS-specific instructions on the fly!

🔑 Concept Focus: Bytecode

Bytecode is **not** machine code! It's a highly optimized set of machine instructions designed specifically to run on an abstract virtual processor (the JVM). This is why Java is completely secure and safe; the host OS is guarded from rogue assembly commands!

DANGER!

⚠️ Common Student Mistake

Do NOT write `javac Demo.class` or `java Demo.class` in your exams!
Remember:
👉 Compile command uses source filename: javac Demo.java
👉 Execution command uses class name ONLY: java Demo

Java Code Structure

Analyzing our very first Java program block line-by-line.

FirstProgram.java

class FirstProgram

{

public static void main(String args[])

{

// This outputs our text to the terminal screen

System.out.println("Welcome to Java");

}

🔎 Code Analysis (Viva Hotspots!)

class FirstProgram

Declares a class template. In Java, ALL executable logic must reside inside a class template! The class name should ideally match the file name.

public

Access specifier meaning the main entry point class must be publicly visible so that the virtual machine (JVM) can trigger execution from outside the directory boundaries!

static

Keyword enabling JVM to load and execute the main process without constructing an object instance of our class first! Saves memory.

void

Method return specification indicating that once the program finishes, it returns absolutely nothing back to the terminal shell environment.

main(...)

Special entry method sequence searched during JVM bootstrap. Accepts external command arguments inside a native String array parameter: String args[].

System.out.println()

Standard standard library print utility:
➔ System: Core system class.
➔ out: Standard console stream object.
➔ println: Prints content followed by a newline.
